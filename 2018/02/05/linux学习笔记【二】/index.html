#linux磁盘系统

##1.基础
cylinder 柱面 sector 扇面 head 磁头

linux中所有硬件设备被抽象为文件，保存在/dev目录下。
SATA,SCSI,SAS,USB等设备名称为sd[a-z]。

  • 命名规则： 设备名称+分区号 （每个区当做独立的磁盘使用）
    主流的分区机制包括：MBR 和 GPT 。
    MPR为传统的bios引导机制，支持分区数量有限，不支持超过2T的硬盘。
  • 分区：主分区 拓展分区（不可用） 逻辑分区
    GPT机制在支持UEFI的硬件上才能使用，系统必须为64bit。
    ##2.FDISK分区工具（基于MNR）
  • fdisk -l 列出所有已安装的硬盘及信息。
  • fdisk /dev/sdn 进行分区 分区之后使用partprobe命令更新分区表

#linux文件系统

##1.基础
创建文件系统的过程就是格式化。
没有文件系统的设备叫做裸(raw)设备。
文件系统的区别在于：日志，分区大小，文件大小，性能。
带有日志的文件系统稳定性较强，出错时可恢复。（写，操，删）但会丧失一定性能。

##2.创建，查看，修复文件系统

###2.1 MKE2FS
mke2fs -t ext4(格式) /dev/sdan 创建文件系统。
参数： - -b blocksize 文件块大小。
-c 检查坏块。
-l label 指定卷标。
-j 建立文件系统日志。

###2.2 MKFS
mkfs ext4 /dev/sdbn
简单，但不能精细化控制。

###2.3 DUMPE2FS
dumpe2fs /dev/sdan 查看文件系统信息。

###2.4 E2LABEL

  • e2label /dev/sdbn 显示标签
  • e2label /dev/sdbn LABEL 加标签
    ###2.5 FSCK
    fsck /dev/sdbn 检查并修复文件系统（必须先卸载）
    -y 直接修复不提示。 -t 文件系统损坏严重时，标识文件系统。
    lost+found 将恢复的文件放入lost+found。
    ##3.挂载，使用文件系统
    创建好文件系统后，需要挂载到一个目录才能使用。linux需要手动挂载。
    系统推荐挂载到/mnt目录下。
  • mount /dev/sdan(要挂载的文件系统) /mnt/（挂载位置）
    mount 不加参数，显示已经挂载的文件系统。
  • umount 文件系统/挂载点 卸载文件系统
    : device is busy 使用 fuser命令查看正在占用设备的进程。
    通过配置文件 /etc/fstab 定义需要自动挂载的文件系统。
    | /dev/sdan | /mnt | ext4 | defaults | 0 0 |
    |———–|——|——|———-|—–|
    | 需要挂载的设备（也可以使用卷标） | 挂载点 | 文件系统 | 挂载选项 | dump fsck |
